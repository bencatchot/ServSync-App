# ServSync Beta Invite Messages

Copy-ready message kit for inviting the first small controlled beta group.

Use these for 2 to 3 trusted contractors and friendly homeowner testers. Keep the tone personal, honest, and clear that this is a private beta.

## Contractor Beta Invite

Subject: Private beta invite for ServSync

Hi [Name],

I'm opening a small private beta for ServSync and wanted to invite you to be one of the first contractors to try it.

ServSync is built to help small contractors keep service work organized: requests, estimates, jobs, reports, invoices, and customer/home records in one place. The goal is not to replace every tool overnight. The goal is to test whether the real workflow feels useful and clear for contractors like you.

For this first beta, I'm only inviting a few trusted contractors and friendly homeowner testers. I'll be hands-on while you try it, and your feedback will shape what gets cleaned up next.

What is ready to test:

- Reviewing service requests.
- Creating and sending estimates.
- Turning accepted estimates into jobs.
- Completing service work or checklist/report jobs.
- Creating and sending invoices.
- Reviewing customer/home work history.
- Sending feedback when something is confusing.

What is not live yet:

- Payments or Stripe checkout.
- QuickBooks/accounting sync.
- Push, text, or email reminder automation.
- Recurring reminders.
- Native mobile apps.
- Full calendar sync.
- Broad public marketplace lead generation.

If you're open to trying it, I'll send over the beta guide and a short first-test checklist.

Thanks,
Ben

## Follow-Up After They Agree

Thanks for being willing to try it.

Here's what I'll send next:

- The beta tester guide.
- Your sign-in/setup steps.
- A short first-test checklist.

For the first pass, please focus on the normal workflow instead of trying to break everything at once:

- Contractor: request -> estimate -> job/report -> invoice.
- Homeowner: request -> estimate review -> invoice -> Home History -> manual reminder.

If something is confusing, broken, or slower than expected, send it over. The most useful feedback is:

- What you were trying to do.
- Which screen you were on.
- What you expected to happen.
- What actually happened.
- Screenshot if easy.

One quick reminder: this is a private beta. Payments, QuickBooks, automatic reminders, native mobile apps, full calendar sync, and broad marketplace lead generation are not live yet.

## What To Test First

Contractor first-test checklist:

- [ ] Review or complete your business profile.
- [ ] Review a service request or customer record.
- [ ] Create an estimate.
- [ ] Send the estimate if the customer is connected.
- [ ] Confirm the accepted-estimate next step is clear.
- [ ] Create a job from an accepted estimate.
- [ ] Complete a service job or checklist/report job.
- [ ] Create and send an invoice where the customer is connected.
- [ ] Send feedback on confusing steps.

Homeowner first-test checklist:

- [ ] Review or add your home/property.
- [ ] Create a service request.
- [ ] Review an estimate.
- [ ] Accept or decline an estimate when appropriate.
- [ ] View an invoice.
- [ ] File an eligible invoice/service record to Home History.
- [ ] Review Home History.
- [ ] Create a manual Home Reminder.
- [ ] Send feedback on confusing steps.

## Known Limitations Wording

Use this wording when sending expectations:

ServSync is in private beta. The current beta is focused on organizing the core service workflow: requests, estimates, jobs/reports, invoices, Home History, and manual reminders.

These are not live yet:

- Payments or Stripe checkout.
- QuickBooks/accounting sync.
- Push notifications.
- Text or email reminder automation.
- Recurring reminders.
- Native mobile apps.
- Full calendar sync.
- Broad public marketplace lead generation.

Home Reminders are manual in-app reminders only. Invoice payment collection happens outside ServSync for now.

## Feedback Request After First Use

Thanks for trying ServSync. When you have a minute, can you send me quick feedback on your first pass?

Helpful questions:

1. What were you trying to do?
2. Where did the workflow feel clear?
3. Where did you get stuck or hesitate?
4. Was anything broken?
5. Was anything too wordy, too hidden, or hard to use on your phone?
6. What is the one thing that would make this more useful for you?

If you ran into an issue, please include:

- Your role: homeowner or contractor.
- The screen or tab.
- What you expected.
- What actually happened.
- Screenshot if easy.

I'm especially looking for honest feedback on whether the request -> estimate -> job/report -> invoice -> Home History flow makes sense.

## Follow-Up If They Have Not Tried It Yet

No pressure, but I wanted to check whether you still want to try the ServSync private beta.

For the first pass, it should be enough to test one small workflow:

- Contractor: review a request or create an estimate/job/invoice.
- Homeowner: create a request, review an estimate/invoice, or try Home History/manual reminders.

If now is not a good time, that's completely fine. I'm keeping the first beta small and will keep improving it as feedback comes in.

## Internal Notes

Use personal judgment before sending:

- Keep invites personal, not mass-marketing.
- Do not imply the app is finished.
- Do not promise lead volume.
- Do not promise payment automation.
- Do not promise automatic reminders.
- Do not ask users to put sensitive payment or private data into feedback messages.
- Route issue details into the feedback triage process.
